1 What is Git primarily classified as?

A. Local Version Control System
B. Centralized Version Control System
C. Relational Database Management System
D. Distributed Version Control System

Correct Answer: Distributed Version Control System

Explanation:

Git is a Distributed Version Control System (DVCS). Unlike centralized systems, every client mirrors the repository, containing the full history of the project.

Incorrect! Try again.

Explanation:

5 What are the three main states that files can reside in within Git?

A. Active, Passive, Archived
B. Created, Updated, Deleted
C. Modified, Staged, Committed
D. Local, Remote, Upstream

Correct Answer: Modified, Staged, Committed

Explanation:

The three main states in the Git lifecycle are Modified (changed but not added), Staged (marked to go into the next commit), and Committed (safely stored in the local database).

18 What is a 'Fast-forward' merge?

A. A merge performed on a remote server only.
B. A merge where the branch pointer simply moves forward because there is no divergent work.
C. A merge that skips conflict resolution.
D. A merge that happens automatically every hour.

Correct Answer: A merge where the branch pointer simply moves forward because there is no divergent work.

Explanation:

If the current branch has not diverged from the branch you are merging in, Git simply moves the pointer forward. This is called a fast-forward merge.

44 What is the primary difference between git reset --hard and git reset --mixed (default)?

A. --hard creates a backup, --mixed deletes files.
B. --hard modifies the working directory, while --mixed keeps working directory changes.
C. There is no difference.
D. --hard changes the commit history, --mixed does not.

Correct Answer: --hard modifies the working directory, while --mixed keeps working directory changes.

Explanation:

git reset --hard resets the index and working directory to the commit state (destroying changes). --mixed resets the index but leaves changes in the working directory.

Correct Answer: A server-side copy of a repository (common in GitHub/GitLab workflows).

Explanation:
