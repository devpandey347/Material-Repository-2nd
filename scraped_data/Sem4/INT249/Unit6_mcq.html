Explanation:

Galera Cluster uses a quorum-based algorithm to ensure data consistency. A cluster partition can continue to operate only if it has a majority of the nodes (quorum). In a cluster with an even number of nodes (e.g., 2 or 4), a network split that divides the nodes evenly (1-1 or 2-2) would cause both partitions to lose quorum and stop accepting writes, leading to downtime. The Galera Arbitrator (garbd) is a lightweight, stateless daemon that acts as a voting member of the cluster without storing a copy of the database. By deploying garbd on a third, independent host, a 2-node cluster becomes a 3-vote cluster. If one DB node gets partitioned, the other node plus the arbitrator can form a majority (2 out of 3 votes), maintain quorum, and continue operating, thus preventing split-brain and improving availability.

Correct Answer: mkfs.xfs -i maxpct=50 /dev/sdc

Explanation:

This question targets a specific, advanced use case for filesystem creation. The -i switch in mkfs.xfs controls inode allocation parameters. The maxpct=<value> attribute for this switch allows the administrator to specify the maximum percentage of the filesystem space that can be allocated to inodes. Setting maxpct=50 reserves up to 50% of the space for the inode table, which is ideal for workloads with millions of very small files. -i size=512 sets the inode size. -m maxpct=50 is incorrect as -m controls the metadata journal size percentage. -i inobpc configures inodes per block cluster, which is another way to control inode density, but maxpct provides a more direct and predictable control over the total space usage for inodes.
