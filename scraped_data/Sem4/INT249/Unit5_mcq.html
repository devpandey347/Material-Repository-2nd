24 You have a file named data.txt and you create a hard link to it named data_hardlink. You also create a symbolic link to it named data_symlink. What happens if you delete the original file, data.txt?

B. The content is preserved and accessible via data_symlink, but data_hardlink becomes a broken link.

D. Both data_hardlink and data_symlink become unusable.

Correct Answer: The content is preserved and accessible via data_hardlink, but data_symlink becomes a broken link.

Explanation:

A hard link is another name for the same inode. The file data is only removed when the inode's link count drops to zero. A symbolic link is just a pointer to a filename; if that file is deleted, the link becomes broken.

42 You are looking for all files in /var/log that are larger than 10MB, have been modified in the last 2 days, and contain the case-insensitive string "fatal error". For maximum efficiency, you want to minimize the number of processes created and avoid running grep on files that don't match the size and time criteria. Which find command is the most efficient and robust?

search for files Hard
A. find /var/log -type f -size +10M -mtime -2 | xargs grep -il "fatal error"
B. find /var/log -type f -size +10M -mtime -2 -print0 | xargs -0 grep -il "fatal error"
C. grep -ril "fatal error" /var/log | xargs -I{} find {} -size +10M -mtime -2
D. find /var/log -type f -size +10M -mtime -2 -exec grep -il "fatal error" {} +

Correct Answer: find /var/log -type f -size +10M -mtime -2 -exec grep -il "fatal error" {} +

Explanation:

This option is the most efficient. find first filters files based on metadata (type, size, mtime), which is very fast. Then, -exec ... {} + gathers all matching filenames and passes them as a single argument list to one grep command (or as few as possible if the list is very long). This minimizes process creation overhead. The xargs versions are similar but less tightly integrated, and the print0/xargs -0 version, while robust against special filenames, is slightly less direct than the -exec ... + POSIX standard. The grep first option is extremely inefficient because it reads the content of every single file before checking metadata.
